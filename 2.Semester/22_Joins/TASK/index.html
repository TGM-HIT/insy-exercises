    INSY GK22 JOIN

    Einführung

    In diesem Kapitel über fortgeschrittene SQL Abfragen wird das Thema JOIN behandelt. In einer Relationalen Datenbank sind die Daten meist über mehrere Tabellen verteilt (um z.B. Redundanzen zu vermeiden, siehe Normalformen). Es kann JOIN verwendet werden um die Daten aus mehreren Tabellen in einer Abfrage zusammenzufügen und auszugeben.

    Ziele

    Die verschiedenen Arten von JOIN kennen und in Verbindung mit den schon bekannten Befehlen verwenden.

    Kompetenzzuordnung

    GK Abfragesprachen

    • JOIN - CROSS, INNER, LEFT, RIGHT, NATURAL, FULL OUTER

    Voraussetzungen

    • Aggregationsfunktionen

    Theorie

    Es wird die flightdatabase verwendet, um das Konzept von JOIN zu verdeutlichen. 

    USE flightdata;
    Das kartesische Produkt zweier Tabellen kann sich wie folgt ausgegeben werden.

    SELECT * FROM airlines, countries; # kartesisches Produkt (CROSS JOIN)
    Dies entspricht einem CROSS JOIN. 
    SELECT * FROM airlines CROSS JOIN countries;
    Dies ist aber in dieser Form wenig hilfreich, weswegen das Ergebnis mit WHERE eingeschränkt werden kann. Dazu werden zwei Spalten die beide Tabellen teilen (Foreign Key) verglichen. Nur wo diese gleich sind, wird die Zeile ausgegeben. Dies entspricht einem INNER JOIN. 

    SELECT * FROM airlines CROSS JOIN countries WHERE airlines.country = countries.code;
    Hier kann auch ON verwendet werden. 
    SELECT * FROM airlines CROSS JOIN countries ON airlines.country = countries.code;
    NATURAL JOIN versucht diese geteilte Spalte selbstständig zu ermitteln (gleicher Name). ON-Clause muss nicht manuell angegeben werden. 
    SELECT * FROM airlines NATURAL JOIN countries;
    Werden mehrere Tabellen in einer Abfrage verwendet, bietet es sich an einen Alias bzw. ein Kurzzeichen zu setzten.

    SELECT * FROM flights f INNER JOIN passengers p ON f.flightnr = p.flightnr;

    Vergleich INNER, LEFT, RIGHT, OUTER JOIN

    SELECT * FROM flights f RIGHT JOIN planes p ON f.planetype = p.id;  #INNER: 190, LEFT: 190, RIGHT: 191
    LEFT JOIN gibt nicht nur die Zeilen aus, wo es genaue Übereinstimmungen gibt, sondern alle Zeilen der Linken Tabelle (NULL Werte für Spalten der rechten Tabelle) 
    SELECT COUNT(*) FROM airlines INNER JOIN countries ON airlines.country = countries.code; # 765


    SELECT COUNT(*) FROM airlines LEFT JOIN countries ON airlines.country = countries.code; # 765

    SELECT * FROM airlines RIGHT JOIN countries ON airlines.country = countries.code; # 832

    In MySQL gibt es kein OUTER JOIN, es kann mit so einem UNION (nächstes Kapitel) gelöst werden.

    SELECT COUNT(*) FROM
(SELECT a.name FROM airlines AS a
LEFT JOIN countries AS c ON a.country = c.code
UNION ALL
SELECT b.id FROM airlines AS b
RIGHT JOIN countries AS d ON b.country = d.code
WHERE b.country IS NULL) AS joined; # 832

    SELECT * FROM airlines
LEFT JOIN countries ON airlines.country = countries.code
UNION ALL
SELECT * FROM airlines
RIGHT JOIN countries ON airlines.country = countries.code
WHERE airlines.country IS NULL;

    Probiere alle Befehle in deiner Datenbank aus.

    Fragestellungen

    Beantworte und dokumentiere folgende Fragestellungen.

    Grundlegend

    • Wofür wird JOIN in einer relationalen Datenbank verwendet?
    • SELECT * FROM table1, table2; Welcher Art von JOIN entspricht diese Abfrage? Welche mathematische Operation wird durchgeführt?
    • Wie lässt sich das Ergebnis der Abfrage oben auf sinnvolle Ergebnisse einschränken?
    • Was hat INNER, LEFT, RIGHT, OUTER JOIN mit der Mengenlehre zu tun?
    • Wann kann NATURAL JOIN verwendet werden?
    • Gibt es in MySQL OUTER JOIN? Mit welchen Befehl kann eines gebaut werden?

    Detaillierte Aufgabenbeschreibung

    Bitte versuche alle wichtigen Information kurz und prägnant zu verschriftlichen.

    Grundanforderungen

    Im Wintersemester hast du dir eine VM mit MariaDB und der flightdatabase installiert. Löse folgende Aufgaben.

    Tipp: Verwende die Diagramm Ansicht in Datagrip: Database Explorer Rechtsklick auf die entsprechende Datenbank --> Diagrams --> Show Diagram

    1) Vorname eines Passagiers der "Ostfriesische Lufttransport" fliegt und auf einem "D" Sitz sitzt. Cruz
    2) Jamaikanische Airline: Airline aus dem Land "Jamaica". Air Jamaica
    3) In welcher Stadt landete der Flug am 25.7.2010 Yaroslavl
    4) Welche tschechische Airline flog in 2009 oder 2010? Travel Service
    5) Letzter Passagier (alphabetisch nach Nachname) der Luxair geflogen ist. Vorname gesucht. Nigel
    6) Nachname eines Passagiers der von Mena nach Reykholar Airport geflogen ist. Vorname ist nur 2 Zeichen lang. Malone
    7) In welchem Land ist der Flug gelandet, mit der Flugnummer die dem Code für "Not Found" entspricht? French Polynesia
    8) Welche Flüge kommen beim Flughafen mit dem Namen "Kingman" an? Es soll eine Tabelle mit den Spalten Fluglinie (Kürzel reicht), Flugnummer und Ankunftszeit zurückgegeben werden. AI,049,2010-05-14 01:06:14 AZ,330,2009-07-08 00:22:14
    9) Von welchen Airlines wird ein Flugzeugtyp geflogen? Es soll eine Tabelle mit den Spalten Hersteller, Typ und Airline zurückgegeben werden, wo es eine Zeile für jede Airline gibt, die den Flugzeugtyp fliegt. Sollte ein Flugzeugtyp von keiner Airline geflogen werden, so soll dieser trotzdem eine Zeile haben, wo die Airline NULL ist. Optional (für ganz engagierte Mitschüler) soll statt der ID der Airline der volle Name zurückgegeben werden.

(Order could be different depending on the order of the JOINs)
    10) Von wo und wohin fliegen die Passagiere? Gib eine Tabelle mit den Spalten Vorname, Nachname, Ausgangsflughafen und Zielflughafen zurück. Beim Ausgangs- und Zielflughafen soll der ausgeschriebene Name und nicht der Airportcode ermittelt werden. Sollte ein Passagier keine gültige Flugnummer haben oder zu einem Airportcode kein gültiger Flughafen eingetragen sein, so sollen Ausgangsflughafen und/oder Zielflughafen NULL sein. Das Ergebnis soll nach dem Zielflughafen absteigend sortiert werden.
